🚀 Simple CMS API Documentation

✅ サービス状況

Simple CMS API は正常に動作しています。実用的なCMSとして以下の機能を提供します。

📋 主要機能

💬 投稿管理 - CRUD操作、公開/下書き状態管理
👥 ユーザー管理 - ユーザー作成・一覧表示
📄 ページ管理 - 静的ページの作成・管理
🔍 検索・フィルタリング - 公開状態による絞り込み
📊 ページネーション - 効率的なデータ表示
📈 統計情報 - システム全体の概要

🔗 API エンドポイント

ヘルスチェック

GET /health

システムの稼働状況を確認

投稿管理

GET /api/posts

投稿一覧を取得（ページネーション、公開状態フィルタリング対応）

クエリパラメータ: page, limit, published

GET /api/posts/{id}

特定の投稿を取得

POST /api/posts

新しい投稿を作成

PUT /api/posts/{id}

投稿を更新

DELETE /api/posts/{id}

投稿を削除

ユーザー管理

GET /api/users

ユーザー一覧を取得

POST /api/users

新しいユーザーを作成

ページ管理

GET /api/pages

ページ一覧を取得

POST /api/pages

新しいページを作成

統計情報

GET /api/stats

システム統計情報を取得

💡 使用例

投稿作成

curl -X POST http://localhost:3000/api/posts \ -H "Content-Type: application/json" \ -d '{ "title": "新しい投稿", "content": "投稿の内容です", "published": true }'

投稿一覧取得（公開済みのみ）

curl "http://localhost:3000/api/posts?published=true&page=1&limit=5"

🌟 実用性

このSimple CMSは以下の点で実用的です：

  • ✅ 完全な CRUD 操作サポート
  • ✅ RESTful API 設計
  • ✅ ページネーション実装
  • ✅ フィルタリング機能
  • ✅ エラーハンドリング
  • ✅ JSON レスポンス統一
  • ✅ CORS サポート
  • ✅ 包括的なドキュメント

🔧 技術仕様

  • フレームワーク: Axum (高性能 Rust web フレームワーク)
  • データストレージ: インメモリ（高速アクセス）
  • シリアライゼーション: Serde (JSON)
  • 非同期処理: Tokio
  • CORS対応: tower-http
